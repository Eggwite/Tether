## Core Data Structures

### PresenceData

Represents the current presence state of a Discord user.

#### Fields

| Field                  | Type                  | Description                                                                                     |
|------------------------|-----------------------|-------------------------------------------------------------------------------------------------|
| `clients`              | object                | An object grouping client information. Contains `active` (array of string) and `primary` (string). |
| `listening_to_spotify` | boolean               | Indicates whether the user is actively listening to Spotify.                                   |
| `spotify`              | object or null        | Present only when `listening_to_spotify` is true. If `listening_to_spotify` is false, `spotify` will be null. |
| `discord_user`         | object               | Basic Discord user identity information.                                                      |
| `status`               | string               | **Allowed values**: `online`, `idle`, `dnd`, `offline`.                                        |
| `activities`           | array of object      | A list of active Discord activities.  |

<Callout title="Note" type="warn">
  The `Activity` object is a partially structured payload derived from Discordâ€™s gateway. Fields may be added over time without a breaking version bump or due to changes in Discord's API. Clients should ignore unknown fields.
</Callout>

### DiscordUser

Represents basic Discord user identity information.

#### Fields

| Field          | Type              | Description                                                                 |
|----------------|-------------------|-----------------------------------------------------------------------------|
| `id`           | string, optional  | The unique identifier for the user.                                        |
| `username`     | string, optional  | The userâ€™s username.                                                       |
| `global_name`  | string, optional  | The userâ€™s global display name.                                            |
| `display_name` | string, optional  | The userâ€™s display name.                                                   |
| `avatar`       | string, optional  | The userâ€™s avatar hash.                                                    |
| `avatar_url`   | string, optional  | The URL to the userâ€™s avatar image.                                        |
| `discriminator`| string, optional  | The userâ€™s discriminator (e.g., `1234`).                                   |
| `bot`          | boolean           | Whether the user is a bot.                                                 |
| `public_flags` | integer           | Public flags associated with the user.                                     |

### Spotify

Represents Spotify listening information.

#### Fields

| Field          | Type              | Description                                                                 |
|----------------|-------------------|-----------------------------------------------------------------------------|
| `track_id`     | string, optional  | The unique identifier for the track.                                       |
| `timestamps`   | object, optional  | The start and end times for the track. **Notes**: Timestamps are in milliseconds since the Unix epoch. |
| `song`         | string, optional  | The name of the song.                                                      |
| `artist`       | string, optional  | The name of the artist.                                                    |
| `album`        | string, optional  | The name of the album.                                                     |
| `album_art_url`| string, optional  | The URL to the album art image.                                            |

  "artist": "Example Artist",
  "album": "Example Album",

  </Tab>
