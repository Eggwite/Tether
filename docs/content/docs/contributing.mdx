# Contributing to Tether

First off, thanks for taking the time to contribute!  
All types of contributions are encouraged and valued. Please read the relevant section below before making your contribution. It helps maintainers and smooths out the experience for everyone.

If you like the project but don’t have time to contribute code, you can still help by:
- ⭐ Starring the [GitHub repo](https://github.com/Eggwite/tether)
- Sharing Tether on social media or with friends
- Mentioning Tether in your own project’s README
- Joining the [Discord server](https://discord.gg/ZggGDhjCyE) to chat, ask questions, or give feedback

## Code of Conduct

This project and everyone participating in it is governed by the [Code of Conduct](https://github.com/Eggwite/tether/blob/main/CODE_OF_CONDUCT.md).  
By participating, you are expected to uphold this code. Please report unacceptable behavior to maintainers.

---

## Asking Questions

Before opening a question, please:
1. Read the [documentation](/docs)
2. Search [GitHub Issues](https://github.com/Eggwite/tether/issues) for similar questions

If you still need help:

3. Open a [GitHub Issue](https://github.com/Eggwite/tether/issues/new)
4. Or ask in the [Discord server](https://discord.gg/ZggGDhjCyE)

Please provide as much context as possible (platform, Node.js version, etc).

---

## Ways to Contribute

- Reporting bugs
- Suggesting features or enhancements
- Improving documentation
- Submitting code via pull requests
- Helping other users in [Issues](https://github.com/Eggwite/tether/issues) or [Discord](https://discord.gg/ZggGDhjCyE)

---

## Reporting Bugs

Before submitting a bug report:
- Make sure you’re using the latest version
- Check the [documentation](/docs) and [existing issues](https://github.com/Eggwite/tether/issues?q=label%3Abug)
- Try to reproduce the issue with a minimal example

When opening a bug report, please include:
- Steps to reproduce
- What you expected to happen
- What actually happened (include error messages, stack traces, etc)
- Your environment (OS, Node.js version, etc)

Open a new issue here: [New Bug Report](https://github.com/Eggwite/tether/issues/new?labels=bug)

---

## Suggesting Enhancements

- Search [existing issues](https://github.com/Eggwite/tether/issues) to see if your idea has already been suggested
- If not, open a [New Feature Request](https://github.com/Eggwite/tether/issues/new?labels=enhancement)
- Clearly describe the enhancement and why it would be useful

---

## Your First Pull Request

1. Fork the repo: [Eggwite/tether](https://github.com/Eggwite/tether)
2. Create a new branch: `git checkout -b my-feature`
3. Make your changes (see [Styleguides](#styleguides))
4. Add/modify tests if needed
5. Commit with a clear message (see [Commit Messages](#commit-messages))
6. Push to your fork and open a [Pull Request](https://github.com/Eggwite/tether/pulls)

We’ll review your PR and may suggest changes. Thank you for contributing!

---

## Improving Documentation

- All docs live in the [`/docs/content`](https://github.com/Eggwite/tether/tree/main/docs/content) folder
- You can edit `.mdx` files directly and submit a PR
- For larger changes, open an issue first to discuss

---

## Styleguides

- Follow the code style used in the repo (see `.editorconfig` and existing files)
- Use clear, descriptive variable and function names
- Write concise, helpful comments where needed

---

## Commit Messages

- Use short, descriptive commit messages (e.g. `fix: handle CORS errors`)
- Reference issues or PRs when relevant (e.g. `closes #42`)

---

## Join the Project Team

If you’re interested in becoming a maintainer or regular contributor, reach out on [Discord](https://discord.gg/ZggGDhjCyE) or open an issue!
